what this is

My first shipped project: this website. It's a portfolio that works like a terminal flying through 3D space — you scroll to travel between waypoints (/home → /projects → /about) instead of scrolling down a page. Everything is one vanilla HTML file. No framework, no build step, no dependencies.

how it works

the 3D camera
The three screens are stacked 1600 units apart in CSS 3D space (translateZ). The page itself is 340vh tall, and a little loop reads how far you've scrolled and pushes the whole world toward the camera. Screens fade and blur as they pass you. Scrolling isn't scrolling — it's throttle.
everything else
The wireframe sphere is just CSS-transformed circles. The HUD (coordinates, depth meter, progress marker) is driven by the same scroll loop. The boot sequence, scanlines and vignette are pure CSS animation. Project cards render from a plain JS array — adding a project is adding one line of data.

what broke (and got fixed)

bug 01 — github pages refused to deploy
The default Pages pipeline failed three times in a row with a generic "try again later". Fix: deleted the Pages site and rebuilt it on the Actions workflow pipeline instead. Now every push deploys automatically.
bug 02 — the missing Y
On phones, "DYLAN_SPIVEY" rendered as "DYLAN_SPIVE" with half a Y. Phone browsers secretly inflate font sizes ("font boosting"), so the text grew wider than its typewriter-animation container. One line of text-size-adjust:100% ended that.
bug 03 — the site loaded into the wrong room
Visitors kept spawning at /projects instead of /home. Browsers restore your last scroll position on reload — and here, scroll position is the camera. The site now forces itself back to depth 0 on every load.
bug 04 — swiping did nothing
The mobile project cards were supposed to be a scrollable row, but touch scrolling silently dies inside 3D-transformed containers. Replaced it with real pagination plus a hand-rolled swipe handler — about six lines of JS.

lessons compiled

Test on a real phone, not just an emulator. The platform can usually do it without a library. And most "weird" bugs have boring, specific causes — you just have to actually look.
